Задание 1. Разработка статического сайта

Создать логически связанную и завершенную информационную среду c использованием фреймов, таблиц или блоков (в соответствии с указанием преподавателя), содержащую перечисленные ниже элементы.
На каждый номерованный элемент должна быть ссылка из этого файла. Пример п.5. Указывать абсолютные адреса.
Защита домашнего задания производится на практических занятиях

  1. Единую для всех страниц навигационную часть, имеющую следующие компоненты:
    1. текстовые и графические гиперссылки, открывающиеся в текущем окне :
    2. в том числе на закладку одной из страниц
    3. на один или несколько внешних сайтов или страниц внешних сайтов, открывающихся в новом окне браузера
    4. для перехода на начальную страницу Вашей среды, открывающуюся в текущем окне браузера
    5. и показывающую посетителю его текущее положение выделением фрагмента навигации (навигацию и выделение оформить с помощью таблицы стилей). http://g06u32.nn2000.info/cgi-bin/py_sql_pages.py?function=page&page_id=20

  2. Информационные страницы (4-6), которые должны содержать:
    1. навигацию (раздел 1, раздел 2, к началу ) внутри страницы с использование закладок
    2. фрагменты, содержащие списки
      • нумерованные
      • не нумерованные
      • вложенные
    3. изображения объектов или субъектов с соответствующими пояснениями и комментариями
    4. изображение большого размера (600 пикселей по горизонтали и 400 по вертикали), открывающиеся в новом окне или закладке в центре как по вертикали, так и по горизонтали в Internet Explorer с использованием миниатуюрной графической гиперссылки (размер по горизонтали до 100 пикселей)
    5. таблицу, содержащую: изображение на заднем фоне, изображение в одной из ячееек, текст, гиперссылки

  3. Работа с данными. C соответствующими ссылками из общей навигации. Для студентов, изучивших базы данных информацию сохранять в базе данных. Для еше не изучивших - сохранять в файлах (*.txt)
    1. страницу (товары, услуги, и др. ) с соответствующими формами для отправки данных на сервер.
    2. страницу для получения контактных данных клиента.
    3. Работающие ссылки на созданные файлы или на веб-интерфейс базы данных, которые должны открываться в новом окне .
